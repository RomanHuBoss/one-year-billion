КРИТИЧЕСКИЕ ИНВАРИАНТЫ

Проверь и при необходимости исправь проект так, чтобы были технически enforced следующие правила:

1. Нет approved non-expired risk_decision_id -> нет order.
2. Нет verified protection -> нет ACTIVE.
3. Strategy module возвращает только SignalCandidate и не импортирует Bybit/execution/order router.
4. Frontend не является source of truth.
5. Stale data -> NO_TRADE/BLOCKED до risk approval.
6. target_equity_usdt / цель 500 -> 1,000,000 используется только в analytics/stress, но не в sizing/risk/execution.
7. Carry/stat-arb Phase 0/1 не имеют live-маршрута исполнения.
8. Manual override может только снижать риск.
9. Live невозможен до Go/No-Go PASS и unresolved CRITICAL/HIGH = 0.
10. Только Bybit V5 category=linear / USDT perpetual/futures. Spot, inverse, options, DCA, martingale, signal bot, copy-trading и любые не-grid/не-breakout стратегии вне разрешенного scope запрещены.

1. Архитектура

Проверь:

- отсутствие бизнес-логики во frontend;

python main.py
python main.py validate
python main.py preflight --mode testnet
python main.py preflight --mode live
python main.py serve --mode testnet
python main.py serve --mode live

Если CLI отсутствует или неполон — реализуй.

2. Data foundation

Проверь и исправь:
- runtime-проверки Bybit instruments;
- category=linear;
- status=Trading;
- tickSize;
- qtyStep;
- minQty;
- minNotional;
- max leverage;

Stale instruments/account/orderbook/funding должны fail-closed блокировать risk approval.

3. Risk engine

Проверь математически и логически:
- расчет risk_usdt;
- stop_distance_abs;
- qty_raw;
- округление qty по qtyStep;
- повторную проверку minQty/minNotional после округления;
- max_loss_if_stop с учетом estimated exit costs;
- effective leverage;
- liquidation distance;
- daily/weekly remaining risk;
- portfolio exposure;
- beta-adjusted exposure;
- expected_net_edge_bps после fees/spread/slippage/funding/safety buffer.

MinNotional не имеет права увеличивать риск выше risk budget. Если минимальный размер ордера нарушает риск — сделка должна быть rejected.

Проверь, что:
- missing stop -> rejected;
- invalid stop -> rejected;
- stale account/specs/orderbook/funding -> rejected;
- no complete cost model -> rejected;
- no positive net edge -> rejected;
- recovery mode не повышает risk;
- target equity не импортируется в risk/sizing/execution;
- risk engine является жестким gate, а не advisory.

4. Execution engine

Проверь:
- deterministic orderLinkId <= 36 characters;
- idempotency table;
- per-symbol lock;
- retry не увеличивает exposure;
- HTTP ack не считается fill;
- private WS + REST reconciliation fallback;
- partial fill handling;
- restart recovery;
- missing protection watchdog;
- reduceOnly exits;
- closeOnTrigger/protection where applicable;
- local/exchange mismatch -> ERROR_RECONCILIATION_REQUIRED;
- unknown exchange position -> symbol freeze + incident + protection/flatten logic.

ACTIVE допустим только при:
- exchange-confirmed position;
- reconciliation PASS;
- protection_state=VALID.

5. Regime classifier

Проверь:
- priority: DE_RISK > NO_TRADE > NEWS_RISK > LIQUIDATION > HIGH_VOL > TREND_UP/TREND_DOWN > RANGE > LOW_VOL;
- hysteresis;
- cooldown;
- safer mixed regime;
- immediate kill-switch;
- permission matrix;
- grid only in RANGE;
- breakout not allowed in RANGE without confirmed shift.

6. Strategies

Проверь:
- strategies возвращают только SignalCandidate;
- candidate содержит strategy, symbol, side, entry_price, stop_price, invalidator, expected edge, required_data, regime_id, feature_id, trace_id, version;
- candidate без stop_price, invalidator, evidence или feature_hash rejected;
- breakout требует Donchian/structure break, volume_z, ATR expansion, BTC alignment, OI/funding sanity;
- micro-grid только в RANGE, с hard stop, max inventory, no add after invalidation;
- carry/funding и pair stat-arb в Phase 0/1 только SHADOW_SIGNAL, без live execution path.

Удалить или исправить любые элементы, превращающие grid в martingale, DCA или усреднение против движения.

7. Математика и эконометрика

Проведи глубокую проверку:
- всех формул доходности;
- compounded return;
- drawdown recovery;
- expectancy_R;
- expected_net_edge_bps;
- PF;
- risk-of-ruin proxy;
- position sizing;
- cost model;
- slippage assumptions;
- funding accounting;
- Monte Carlo;
- walk-forward validation;
- leakage protection;
- label generation;
- timestamp alignment;
- same-bar TP/SL ambiguity;
- feature timestamp <= decision timestamp.

Проверь, что:
- backtest без costs считается invalid;
- gross-only results не используются как основание для live;
- рост costs монотонно снижает expected_net_edge;
- случайное перемешивание временных рядов запрещено;
- будущие high/low/funding/volume не попадают в features;
- incomplete bars не используются для decision;
- ML не открывает сделки, не считает size и не меняет leverage;
- stale/missing/drifted model -> BLOCK/UNAVAILABLE fail-closed.

Важно:
не делай вид, что live-ready подтвержден, если в песочнице невозможно проверить:
- реальные Bybit keys;
- реальные permissions;

В таком случае проект должен быть technical-live-ready / live-gated:

- unsafe live невозможен.
